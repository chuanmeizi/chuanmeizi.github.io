搭建个人博客教程

搭建个人博客教程

一.基础环境配置

1.安装node
node一款运行效率非常高的web服务器，采用javascript语法
注意：建议安装版本为 6.x
测试环境：
windows+r 回车 输入cmd 回车，输入node -v

2.安装git
git：版本控制器，团队协作开发的时候使用

二.安装hexo脚手架

切换镜像（切换服务器）：
npm install -g cnpm –registry=https://registry.npm.taobao.org
安装：
cnpm install -g hexo-cli

命令： npm install hexo-cli -g //（注释）在全局安装hexo脚手架
从node服务器下载hexo脚手架到本机
node服务器在国外 怎么办？

三.创建博客项目

1.桌面新建名为blog的文件夹
命令：mkdir blog
2.进入blog文件夹
命令：cd blog
3.初始化博客项目
命令：hexo init
4.导入npm依赖包
命令：cnpm install （简写：cnpm i）

5.启动hexo项目服务器
命令：hexo s
启动之后可以在；浏览器地址栏输入：localhost:4000预览博客
6.编译博客
每当博客内容有更改的时候，需要向编译再刷新浏览器
命令：hexo g

四.git常用命令

1.在任意位置右键选择git bash here 可以在cmd打开当前路径
2.路径切换：
返回上级目录：cd …
进入下级目录：cd 路径名称
3.显示当前路径的文件/夹列表：ls
4.创建文件夹：mkdir 文件夹名
5.切换盘符：
如：当前在c盘某路径，需要进入d盘某路径
.d：回车（输入d冒号空格然后回车）
.cd 详细路径

换博客主题：
vs-code先打开 hexo s
再去开网址localhost：4000

五、gitHub的使用

1、本地创建ssh key
桌面右键git bash here，然后输入ssh-keygen -t rsa -C “邮箱”
记住ssh key的保存路径，然后一路回车，直到显示很多小圆圈
默认路径：c:/users/用户名/.ssh
2、打开c:/users/用户名/.ssh文件夹下的id_rsa.pub文件(打开方式选择记事本)
ctrl+a全选内容，然后ctrl+c复制备用
3、打开github.com登录个人账号，点击头像右侧小三角在下拉菜单里面选择settings
在新页面左侧选择ssh and gpg keys，然后点击右侧绿色new ssh key按钮
title：随意填
key：ctrl+v粘贴刚才复制的key，然后点击下方绿色add ssh key按钮
4、回到git命令行工具，输入ssh -T git@github.com回车
如果提示Hi XXX! You’ve successfully authenticated, but GitHub does not provide shell access.代表添加key成功
5、验证github邮箱和用户名：
在git命令行窗口输入：
git config –global user.name “用户名” 回车（没有任何提示代表验证成功）
git config –global user.email “邮箱” 回车
注意：编写命令的时候注意英文符号以及必要的空格
6、github创建仓库
Github页面点击头像左侧加号，点击下拉列表第一个按钮new repository
填写Repository name，就是仓库的名字，尽量保证是英文
点击下方create repository按钮
7、桌面右键git bash here，然后copy如下命令：
echo “测试” >> README.md #在桌面创建内容为“测试”的md文件
git init #初始化git
git add README.md #添加刚才生成的md文件
git commit -m “添加 README.md 文件” #编写更新说明
git remote add origin git@github.com:用户名/仓库名.git #设置需要提交的仓库
git push -u origin master #向仓库推送文件
注意：最后一步首次推送需要验证用户名密码
第一次推送成功后，后续还需要推送文件到之前仓库怎么办？三步：
git add 文件名.后缀
Git commit -m “更新说明”
git push -u origin master

六.代码托管到github

1·在博客项根目录的_config.yml 文件的底部添加如下信息
deploy：
type：git
repository：https//github.com/用户名/用户名.github.io
branch:master
注意：格式的缩进跟_config.yml里面其他项保持一致
2·添加git依赖包
命令行执行：npm install hexo-deployer-git –save（执行一次）
3·执行：hexo g 编辑博客项目
4·执行hexo d 将代码托管到github
如果成功，可以通过 用户名.github.io访问个人博客

七、域名解析

记录类型    主机记录    解析线路(isp)    记录值
CNAME          www             默认           用户名.github.io
A                   @                 默认           192.30.252.153
A                   @                 默认           192.30.252.154

八.域名绑定到个人博客

在public文件夹下添加CNAME文件（不带后缀名）
在CNAME 里面填入更让人域名（不带www）
